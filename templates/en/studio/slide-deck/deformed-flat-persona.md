# Deformed Flat Persona

**Source:** Kawai
**Format:** Slide Deck
**Difficulty:** Beginner

## Prompt

```
# presentation_design_spec_deformed_flat_persona.yaml
# Style: Deformed Flat Persona
# Concept: "Character-Led Communication — Human Warmth through Illustration"

Global Design Settings:
  Tone: "Friendly, approachable, warm, clear, character-driven storytelling"

  Color Palette:
    Base: "One solid flat pastel color per slide — rotated across the deck"
    PastelPalette:
      - "#FFE0B2 (warm peach)"
      - "#E8F5E9 (mint green)"
      - "#E3F2FD (sky blue)"
      - "#F3E5F5 (soft lavender)"
      - "#FFF9C4 (pale yellow)"
    Text: "#2D2D2D (warm charcoal — not harsh black)"
    Outline: "#1A1A1A (thick friendly ink black — all character and illustration outlines)"
    Rule: "Maximum 3 colors per slide: background + character fill(s) + one accent. No exceptions."

  Typography:
    Headings: "Rounded, friendly bold sans serif — approachable, not corporate."
    Body: "Clean rounded sans serif. Small, well-spaced."
    Numbers: "Bold, large, playful — treated as visual characters."
    Language: "The language should be what users said in the prompt."

  Common Layout Rules:
    Characters: "All human figures use deformed proportions: large head (50–60% of total body height), small rounded body, simplified limbs."
    Outlines: "All characters and illustration elements use thick black outlines (4–6pt). Never thin or hairline."
    ColorCount: "Strict maximum 3 flat colors per slide. No gradients, no textures, no drop shadows."
    Expressions: "Characters always show clear, exaggerated facial expressions — happy, surprised, thinking, explaining. No neutral faces."
    BackgroundRotation: "Change the pastel background color from slide to slide to create rhythm and visual flow."

Layout Variations:
  - Type: "Introduction / Title"
    Design: "Solid pastel background. A friendly deformed character in the center-right, waving or gesturing toward the title. Title in large rounded bold type on the left. Small speech bubble from the character with a short welcoming phrase."

  - Type: "Persona Profile"
    Design: "Clean card layout on pastel background. Large character illustration on the left (occupying ~40% of slide width). Name in bold headline, role or description in body text on the right. Small icon tags for attributes listed below."

  - Type: "Explainer with Character"
    Design: "Character stands beside or inside a process diagram. Speech bubbles or thought clouds contain key explanations. Character pose and expression reflects the content (thinking = hand on chin, explaining = pointing upward)."

  - Type: "Step-by-Step Guide"
    Design: "3–4 steps shown horizontally. Each step: a numbered circle, a small character in the relevant pose, a bold step title, and 1–2 lines of text. Character expressions change step to step to show progression."

  - Type: "Before / After"
    Design: "Slide split vertically. Left (before): character in frustrated or negative pose with a muted or darker-toned color. Right (after): character in happy or celebratory pose with a brighter pastel. Simple arrow transition between panels."

  - Type: "Team / Cast"
    Design: "Multiple characters arranged in a horizontal row, each slightly different in color or costume detail. Name and role below each character. Equal spacing. Feels like a cast introduction or team page."

  - Type: "Data with Emotion"
    Design: "A large number or statistic displayed prominently. A character reacts to it — jumping if positive, slumping if negative, pondering if complex. Brief text explanation beside the character."

Design Rules:
  - "Head-Body Ratio: Character heads are always 50–60% of total body height. This is the core deformed aesthetic."
  - "Outline Weight: All illustration outlines are 4–6pt thick. Never thin or hairline."
  - "Three-Color Maximum: Background + character color(s) + one accent = maximum 3 colors. No exceptions."
  - "No Gradients: All fills are 100% flat solid color. No shading, no gradients, no drop shadows."
  - "Expressions Matter: A neutral-faced character is incorrect. Every character must show an intentional, clear emotion."
  - "Generation: Generate a cohesive multi-slide presentation where deformed flat characters guide the viewer through the content. Maintain consistent character proportion, outline weight, and flat-color discipline throughout."
```
